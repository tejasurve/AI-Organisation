# SaaS Landing Page demo

A runnable SaaS landing page with a working signup form, built around the AI-organisation pipeline output.

## Run it

```bash
cd examples/saas-landing-page
node server.mjs
```

Then open <http://localhost:3000> in your browser. Use a custom port with `PORT=3737 node server.mjs`.

No `npm install`. No build step. Requires Node 22+ (ESM + `node:` imports + `crypto.randomUUID`).

## What you'll see

- Sticky top nav with brand mark + nav links
- Hero section: "An entire engineering team. Powered by AI." + primary CTA
- 3 feature cards (Strategic thinking / Atomic execution / Audit trail)
- "How it works" — the 9-step pipeline (CEO → … → file writer)
- Email signup section that POSTs to `/api/signup`
- Footer

Try the form with:

- a real-looking email → `Thanks — you're on the list.`
- the same email again → `Looks like you're already on the list…` (deduplication)
- a malformed email → inline `That doesn't look like a valid email` error

You can inspect the in-memory store at <http://localhost:3000/api/signups>.

## Upgrade path: Postgres + Drizzle (the CTO's actual stack)

The demo uses an in-memory `store.mjs` with the same row shape as the AI-generated Drizzle schema. To upgrade:

1. `npm install drizzle-orm postgres` in this folder.
2. Replace `store.mjs` with a Drizzle client that uses `generated/t-signup-storage-1/lib/db/schema/signups.ts`.
3. Run `drizzle-kit generate` to produce the SQL migration (the developer agent's notes say this step is delegated to the build pipeline, not to the developer agent itself).
4. Point `DATABASE_URL` at a Neon / local Postgres instance.

Or — closer to the spirit of this project — run the remaining tasks (`t-signup-storage-2`, `t-landing-page-1`, `t-landing-page-3`) through the AI-organisation pipeline, and replace the corresponding files in this demo with the generated outputs.
